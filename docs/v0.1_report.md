# AgentGym v0.1 Report

Date: 2026-05-16

AgentGym v0.1 is a small executable benchmark for coding agents. The current version contains four synthetic Python API tasks, a local CLI, public tests, hidden tests, task validation, suite execution, and reference patches.

The purpose of this release is not breadth. It is to prove the task format and show that small repo tasks can isolate useful coding-agent failure modes.

## Benchmark Snapshot

| Metric | Result |
| --- | --- |
| Tasks | 4 |
| Language | Python |
| Domain | API validation and serialization |
| Public test result on starter tasks | 4/4 pass |
| Hidden test result on starter tasks | 0/4 pass |
| Runner | Local CLI |
| Agent invocation | Not included yet |

The expected baseline is that each starter repository passes public tests and fails hidden tests. That means the visible behavior is plausible, but the stricter edge case is still broken.

## Current Tasks

| Task | Failure mode | Hidden behavior checked |
| --- | --- | --- |
| `python-api-001` | Input normalization | Whitespace-only required fields are rejected. |
| `python-api-002` | Strict nested type validation | String and integer preference values are rejected instead of coerced. |
| `python-api-003` | Cross-field validation | Account type and tax ID rules are enforced together. |
| `python-api-004` | Response serialization allowlisting | Internal user fields are not leaked in API responses. |

These tasks are intentionally small. The goal is for a human reviewer to understand each task quickly and for an agent failure to map to one clear engineering mistake.

## CLI Result

The current suite result on unmodified starter tasks is:

```text
id              public  hidden  status
python-api-001  pass    fail    fail
python-api-002  pass    fail    fail
python-api-003  pass    fail    fail
python-api-004  pass    fail    fail

4/4 tasks passed public tests
0/4 tasks passed hidden tests
```

This is the intended result. The starter tasks are not supposed to pass hidden tests until an agent or developer fixes the bug.

## What This Shows

The first four tasks show that shallow patches can look correct against public tests while missing stricter behavior:

- validation can happen in the wrong order
- type coercion can silently violate an API contract
- fields can be valid individually but invalid together
- response builders can leak private data if they copy internal records

These are small examples, but they are real classes of bugs.

## What This Does Not Show Yet

AgentGym v0.1 does not yet measure frontier model performance. It also does not yet provide:

- private held-out tasks
- Docker isolation
- leaderboard scoring
- automatic agent invocation
- broad language or framework coverage

The current repo is a public method and task-format proof, not a final benchmark.

## Next Milestones

The next useful milestones are:

1. Expand to 6-8 Python API tasks while keeping one failure mode per task.
2. Add suite summaries grouped by failure mode.
3. Add an optional agent command hook for running external coding agents against tasks.
4. Decide whether to create private or generated task packs for cleaner evaluation.

The main constraint should stay the same: every task must be executable, inspectable, deterministic, and backed by a reference patch.
